**APIClient.swift** — struct con todas las llamadas HTTP:

Métodos requeridos:
1. `func login(password: String) async throws -> String` — POST /auth/login, body JSON {"password": pw}, decodifica LoginResponse con campo "token"
2. `func renewToken(current: String) async throws -> String` — POST /auth/renew con Authorization Bearer, misma respuesta
3. `func uploadAudio(fileData: Data, fileName: String, mimeType: String, token: String) async throws -> String` — POST /process multipart, devuelve job_id
4. `func pollJobStatus(jobId: String, token: String) async throws -> JobResult` — bucle GET /result/{id} cada 3s, máximo 60 intentos
5. `func getUsage(token: String) async throws -> UsageResponse` — GET /usage

Modelos de respuesta (en APIClient.swift o fichero separado APIModels.swift según preferencia):
```swift
struct LoginResponse: Decodable { let token: String }
struct ProcessResponse: Decodable { let job_id: String }
struct JobStatusResponse: Decodable { let status: String; let result: JobResult? }
struct JobResult: Decodable { /* stems, lyrics, chords — placeholder para Phase 6 */ }
struct UsageResponse: Decodable { let month: String; let songs_processed: Int; let gpu_seconds: Double; let estimated_cost_usd: Double }
```

Helper privado `func makeRequest(endpoint:token:body:contentType:) -> URLRequest` — construye URLRequest inyectando Authorization: Bearer si token != nil.

Helper privado `func checkResponse(_ response: URLResponse) throws` — extrae HTTPURLResponse, lanza .unauthorized en 401, .httpError(code) en otros no-2xx.

Multipart upload: usar `MultipartRequest` struct con boundary UUID, métodos `addFile` y campo Content-Type correcto. El boundary del header debe coincidir exactamente con el del body.

Polling: usar `Task.checkCancellation()` en cada iteración antes del sleep. Detectar status que empieza por "error:" → lanza `processingFailed`. Detectar status == "completed" → devuelve result.

Tests: crear `StrataClientTests/NetworkTests.swift` con URLProtocol mock que intercepta llamadas. No se conecta al servidor real. Tests cubren todos los comportamientos del <behavior>.
